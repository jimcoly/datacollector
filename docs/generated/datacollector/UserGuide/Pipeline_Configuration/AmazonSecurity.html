Security in Amazon Stages

You can configure Amazon stages to use one of the following authentication methods to connect securely to Amazon Web Services (AWS):

Instance profile
When Data Collector runs on an Amazon EC2 instance that has an associated instance profile, Data Collector uses the instance profile credentials to automatically authenticate with AWS.
If the IAM policies attached to the instance profile do not grant the required access, you can configure the stage to assume another role when connecting to AWS.
For more information about associating an instance profile with an EC2 instance, see the Amazon EC2 documentation.
AWS access keys
When Data Collector does not run on an Amazon EC2 instance or when the EC2 instance doesn’t have an instance profile, you can authenticate using an AWS access key pair. When using an AWS access key pair, you specify the access key ID and secret access key to use.
Note: When configuring an Amazon S3 stage or an Amazon SQS Consumer origin to access a public bucket, you can configure the stage to connect anonymously using no authentication.

Assume Role for Instance Profile Authentication

When using instance profile authentication, you can configure an Amazon stage to assume another IAM role.

For example, if the IAM policies attached to the instance profile do not grant access to write to Amazon S3 resources, you can configure the Amazon S3 destination to assume another role that does grant write access.

When an Amazon stage assumes a role, it temporarily gives up the instance profile role permissions and uses the permissions assigned to the assumed role. To assume a role, the stage calls the AWS STS AssumeRole API operation and passes the role to use. The operation creates a new session with the temporary credentials, as long as the instance profile has permission to assume the specified role.

You can configure an Amazon stage to assume a role in the following ways:
Assume a role with no restrictions

When configured to assume a role with no restrictions, any user who starts the pipeline can assume the role specified in the Amazon stage.

For example, any user who starts the pipeline can assume the finance role when the instance profile has permission to assume the finance role.

Assume a role using session tags to restrict role access
For increased security, you can configure a stage to assume a role and set session tags to restrict the users allowed to assume the role. When configured to set session tags, the stage passes the following session tag to the AWS STS AssumeRole API operation:

streamsets/principal=<user>

Where <user> is the name of the currently logged in Data Collector user that starts the pipeline or the Control Hub user that starts the job for the pipeline. For a Control Hub user, the stage uses an abbreviated version of the Control Hub user ID, ignoring "@<organization ID>". For example, using joe instead of joe@MyCompany as the user name.

The instance profile verifies that the user set in the session tag can assume the specified role.

For example, when user Joe starts the pipeline, he can assume the finance role when the instance profile has permission to assume the finance role and when the instance profile allows the user joe to assume the finance role. The user Emily cannot assume the finance role because the IAM policy for the instance profile does not allow the user emily to assume the finance role.

To configure an Amazon stage to assume a role, you first must define the IAM policy in AWS that allows the instance profile to assume a role. Then, you configure the required stage properties in Data Collector.

Define the IAM Policy

In AWS, define an IAM policy that allows the instance profile to assume a role.

The policy that you define depends on whether you want to allow stages to assume a role with or without restrictions:
IAM policy to assume a role with no restrictions
Define an IAM policy that allows the instance profile to assume a role.
For example, you might define the following policy where:
  • <account_id> is the ID of your AWS account.
  • <role_name> is the name of the role to assume. 
{
 "Version": "2012-10-17",
 "Statement": [
   {
     "Sid": "",
     "Effect": "Allow",
     "Principal": {
       "AWS": "arn:aws:iam::<account_id>:role/<role_name>"
     },
     "Action": [
       "sts:AssumeRole"
     ]
   }
 ]
}
IAM policy to assume a role using session tags to restrict role access
Define an IAM policy that allows the instance profile to assume a role, use session tags, and restrict the session tag values to specific users.
For example, you might define the following policy where:
  • <account_id> is the ID of your AWS account.
  • <role_name> is the name of the role to assume.
  • <user1> and <user2> are Data Collector or Control Hub user accounts allowed to assume the role. To define a Control Hub user account, use an abbreviated version of the Control Hub user ID, ignoring "@<organization ID>". For example, using joe instead of joe@MyCompany as the user name. 
{
 "Version": "2012-10-17",
 "Statement": [
   {
     "Sid": "",
     "Effect": "Allow",
     "Principal": {
       "AWS": "arn:aws:iam::<account_id>:role/<role_name>"
     },
     "Action": [
       "sts:AssumeRole",
       "sts:TagSession"
     ],
     "Condition": {
       "StringEquals": {
         "aws:RequestTag/streamsets/principal": ["<user1>", "<user2>"]
       }
     }
   }
 ]
}

For more information about defining an IAM policy, see the Amazon IAM documentation.

Configure Stages to Assume a Role

After you define an IAM policy in AWS that allows the instance profile to assume a role, you can configure Amazon stages to assume the role.

  1. On the primary tab of the Amazon stage, select Instance Profile for the Authentication Method property.
  2. Select Assume Role.
  3. Configure the following properties:
    Assume Role Property Description
    Role ARN Amazon resource name (ARN) of the role to assume, entered in the following format:

    arn:aws:iam::<account_id>:role/<role_name>

    Where <account_id> is the ID of your AWS account and <role_name> is the name of the role to assume.

    The role ARN must match the role ARN specified in the IAM policy defined for the instance profile.
    Role Session Name Optional name for the session created by assuming a role. Overrides the default unique identifier for the session.
    Session Timeout Maximum number of seconds for each session created by assuming a role. The session is refreshed if the pipeline continues to run for longer than this amount of time.

    Set to a value between 3,600 seconds and 43,200 seconds.

    Set Session Tags Sets a session tag to record the name of the currently logged in Data Collector user that starts the pipeline or the Control Hub user that starts the job for the pipeline. The instance profile verifies that the user set in the session tag can assume the specified role.

    Select only when the IAM policy defined for the instance profile uses session tags and restricts the session tag values to specific users.

    When cleared, the stage does not set a session tag. As a result, any user who starts the pipeline can assume the specified role.
